Хочу сделать свой проект CRM, который можно будет использовать в разных проектах. Есть репозиторий и название проекта eikiei.

Вот его стек:

0. Проект на TypeScript

1. FRONTEND для web
1.1 Next.js через create next-app (публичные страницы (SSR/SSG), CRM (CSR)) # Версии - "next": "15.5.7", "react": "19.2.1", устновка pnpm create next-app web из папки apps
1.2 Zustand для UI/auth состояний
1.3 TanStack Query для работы с серверным состоянием, кешированием
1.4 Tailwind CSS для стилей
1.5 Storybook для документация компонентов

2. BACKEND
2.1 Fastify для API сервера
2.2 Prisma для работы с БД (обязательная версия ^7.x.x)
2.3 Zod для валидации
2.4 Pino для логирования

3. ДАННЫЕ
3.1 PostgreSQL основная БД
3.2 Redis кеш, сессии, pub/sub

4. REALTIME
4.1 Socket.IO для чата и уведомлений

5. BACKGROUND JOBS
5.1 BullMQ очереди задач

6. FILE STORAGE
6.1 локальная файловая система или MinIO для прода

7. File Manager
7.1 загрузка файлов, папки, права доступа, привязка к пользователям

8. Авторизация
8.1 JWT + refresh tokens

9. ТЕСТЫ
9.1 Jest unit тесты
9.3 Playwright e2e тесты

10. Docker инфраструктура

11. Email через SMTP




Структура проекта в FSD
./
	/apps/       # приложения
		web/     # фронтенд
		api/     # бэкенд
	/packages/   # общие пакеты (shareable code)
		config/  # конфигурации (env, prisma, logger)
		shared/  # общие функции/утилиты
		ui/      # UI-компоненты для фронтенда
	docker-compose.yml
	.gitignore # только в корне
	.dockerignore # если нужен
	.package.json # если нужен
	.dockerignore # если нужен


Помоги начать, нужно настроить минимально рабочий вариант, чтобы был docker, который запускал бы все необходимое. Вначале минимально рабочий api и уже установлен next через create next-app. Не использовать pnpm, только npm. Обязательный hot-reload, и отсутствие пересборки контейнеров каждый раз. Не использовать init.sh





На вкладке General:
Name: eikiei_postgres_local (можно любое)
На вкладке Connection:
Host name/address: postgres (имя сервиса из docker-compose.yml)
Port: 5432
Username: crm_user
Password: crm_password
Maintenance database: crm_db
